# 指标定义表

下面整理的是当前项目设计报告里最可能用到的指标，包括旧框架和新框架两部分。  
表格中的“计算逻辑”强调**怎么得到这个数**，“含义”强调**这个数在实验上说明什么**。

## 新框架指标

| 指标名 | 计算逻辑 | 直观含义 | 优化方向 | 备注 |
| --- | --- | --- | --- | --- |
| `train_mse` | 训练集上，预测 latent `pred_z` 与目标 latent `z_gt` 的均方误差，公式可理解为 `mean((pred_z - z_gt)^2)` | 图像/文本编码器预测出的 DeepCAD latent 与目标 latent 有多接近 | 越小越好 | 这是新框架最核心的训练损失 |
| `test_mse` | 测试集上，同样计算 `pred_z` 与 `z_gt` 的均方误差 | 模型在未见测试样本上，latent 回归是否准确 | 越小越好 | 用于选取 `best.pt` |
| `train_cosine` | 训练集上，`pred_z` 与 `z_gt` 的余弦相似度平均值 | 预测 latent 与目标 latent 的方向是否一致 | 越大越好 | 对尺度不敏感，更强调方向一致性 |
| `test_cosine` | 测试集上，`pred_z` 与 `z_gt` 的余弦相似度平均值 | 测试样本上的 latent 方向是否对齐 | 越大越好 | 和 `test_mse` 互补 |
| `cmd_token_acc` | 将解码后的 CAD 序列与 GT 序列按位置对齐，仅比较每一步的命令类型（如 `Line/Arc/Circle/EOS/SOL/Ext`）是否一致，再对所有 token 取平均 | 命令类别预测得准不准 | 越大越好 | 只看命令，不看参数 |
| `token_exact_acc` | 按位置对齐后，要求一个 token 的**命令和该 token 所有参数**都完全一致，统计逐 token 的平均正确率 | 单步 CAD token 是否完整预测正确 | 越大越好 | 比 `cmd_token_acc` 更严格 |
| `sequence_cmd_exact_rate` | 一条序列中，所有 token 的命令都预测正确，记为 1，否则记为 0；再对全部样本取平均 | 一整个 CAD 序列的命令结构是否完全正确 | 越大越好 | 比 `cmd_token_acc` 严格很多 |
| `sequence_exact_rate` | 一条序列中，所有 token 的命令和参数都完全正确，记为 1，否则记为 0；再对全部样本取平均 | 是否把整个 CAD 序列完整无误地复原出来 | 越大越好 | 当前最严格的序列级离散指标之一 |
| `mean_pred_len` | 预测序列长度的平均值 | 模型通常会输出多长的 CAD 序列 | 不单独追求大/小 | 需要结合 GT 长度看 |
| `mean_gt_len` | GT 序列长度的平均值 | 数据集目标序列通常有多长 | 不适用 | 主要作为参考量 |
| `mean_len_abs_error` | 对每个样本，计算 `abs(pred_len - gt_len)`，再取平均 | 预测长度与真实长度差多少 | 越小越好 | 长度偏差过大常意味着结构错误 |
| `first_divergence_missing_rate` | 序列中如果完全没有分歧，则 `first_divergence_step = -1`；这个指标统计 `first_divergence_step = -1` 的比例 | 有多少样本从头到尾都没有离散层面的分歧 | 越大越好 | 在当前脚本中，它与 `sequence_exact_rate` 数值一致 |
| `pred_solid_valid_rate` | 对预测序列调用 DeepCAD/OCC 的实体构造与合法性检查；能成功构造有效实体记为 1，否则记为 0，最后取平均 | 预测结果里有多少真的能变成合法 CAD 实体 | 越大越好 | 对答辩最重要的工程型指标之一 |
| `gt_solid_valid_rate` | 对 GT 序列做同样的实体合法性检查，最后取平均 | 数据本身能成功导出实体的比例 | 越大越好 | 一般接近 1，但不一定严格等于 1 |

## 新框架长度分桶指标

| 指标名 | 计算逻辑 | 直观含义 | 优化方向 | 备注 |
| --- | --- | --- | --- | --- |
| `len_01_06` | GT 序列长度在 1 到 6 之间的样本集合 | 极短序列 / 简单样本 | 不适用 | 常对应简单草图或简单实体 |
| `len_07_12` | GT 序列长度在 7 到 12 之间的样本集合 | 短到中等复杂样本 | 不适用 | 通常是最容易体现改进的一档 |
| `len_13_24` | GT 序列长度在 13 到 24 之间的样本集合 | 中等复杂样本 | 不适用 | 是当前系统的重要挑战区 |
| `len_25_plus` | GT 序列长度大于等于 25 的样本集合 | 长序列 / 高复杂样本 | 不适用 | 当前最难的一档 |
| `by_bucket.*.sequence_exact_rate` | 在某个长度分桶内，计算 `sequence_exact_rate` | 某一复杂度范围内，整序列完全正确的比例 | 越大越好 | 用来分析模型对简单/复杂样本的适应能力 |
| `by_bucket.*.pred_solid_valid_rate` | 在某个长度分桶内，计算 `pred_solid_valid_rate` | 某一复杂度范围内，可生成合法实体的比例 | 越大越好 | 很适合放到对比图里 |
| `by_bucket.*.mean_len_abs_error` | 在某个长度分桶内，计算 `mean_len_abs_error` | 某一复杂度范围内，长度偏差大小 | 越小越好 | 能帮助解释模型是在“截短”还是“过长生成” |

## 检索/融合推理模式相关概念

| 名称 | 计算逻辑 | 直观含义 | 备注 |
| --- | --- | --- | --- |
| `direct` | 直接使用模型预测的 latent：`final_z = pred_z` | 完全信任模型自己的生成结果 | 最“纯”的生成模式 |
| `nearest` | 在训练集 latent 库中检索最近邻，并直接使用该 latent：`final_z = nearest_z` | 完全信任检索到的相似样本 | 更像检索增强，不太像纯生成 |
| `blend(alpha)` | 线性融合：`final_z = alpha * pred_z + (1 - alpha) * nearest_z` | 在“模型自己预测”和“训练集相似样本”之间折中 | `alpha` 越大越信任模型，越小越信任检索 |
| `blend_alpha` | `blend` 模式中的权重系数 | 决定最终 latent 有多少来自模型预测，有多少来自最近邻 | `alpha=1` 近似 `direct`，`alpha=0` 近似 `nearest` |
| `retrieval_score` | 检索时得到的相似度或距离分数 | 检索到的最近邻与当前预测 latent 有多接近 | 分数解释取决于具体检索度量（当前可为 cosine 或 l2） |

## 旧框架指标

下面这些指标来自旧的自回归双模态框架，主要从 TensorBoard 标量中读取。

| 指标名 | 计算逻辑 | 直观含义 | 优化方向 | 备注 |
| --- | --- | --- | --- | --- |
| `train/loss` | 训练阶段总损失，通常由命令损失和参数损失加权组成 | 训练是否在收敛 | 越小越好 | 旧框架的主损失 |
| `train/cmd_loss` | 训练阶段命令分类损失 | 命令 token 预测误差 | 越小越好 | 对应命令头 |
| `train/param_loss` | 训练阶段参数分类损失 | CAD 参数离散分类误差 | 越小越好 | 对应参数头 |
| `val/loss` | 验证集上的总损失 | 模型在 teacher-forced 条件下的整体泛化情况 | 越小越好 | 旧框架中最常看的验证指标之一 |
| `val/cmd_token_acc` | 验证集上，仅看命令 token 的逐位置准确率 | 命令分类是否准确 | 越大越好 | 通常会较高 |
| `val/param_token_acc` | 验证集上，有效参数位置的逐位置准确率 | 参数离散分类是否准确 | 越大越好 | 一般比命令更难 |
| `val/token_exact_acc` | 单个 token 的命令和参数都完全正确的比例 | 单步 CAD token 是否完整预测正确 | 越大越好 | 比单纯命令准确率严格 |
| `val/sequence_cmd_exact_acc` | 一整条序列的命令都完全正确的比例 | 序列级命令结构是否完全正确 | 越大越好 | 旧框架里这个指标曾出现实现 bug，解读要谨慎 |
| `val/sequence_exact_acc` | 一整条序列的命令和参数都完全正确的比例 | teacher-forced 条件下的整序列精确率 | 越大越好 | 尽管严格，但仍不等于真实自由生成质量 |

## 报告撰写时的解释建议

| 指标 | 报告里建议怎么解释 |
| --- | --- |
| `test_mse` / `test_cosine` | 说明 latent 回归是否成功，是新框架训练稳定性的直接证据 |
| `token_exact_acc` / `sequence_exact_rate` | 说明离散 CAD 序列层面的复原质量 |
| `pred_solid_valid_rate` | 说明生成结果是否真正能成为合法 CAD 模型，是最工程化、最容易让老师理解的指标 |
| 长度分桶指标 | 说明模型在简单样本和复杂样本上的能力差异，帮助分析当前系统的瓶颈 |
| 旧框架的 `val/*` 指标 | 主要用于说明旧框架虽然训练曲线看起来不错，但不能保证真实推理稳定性，从而支撑框架切换的必要性 |
